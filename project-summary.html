Summer Ngo's M1 Project Proposal

Project Description

Topic: My website is going to be a potential social media-like platform for artists to share their artwork, along with ease of creating a community for artists.

Audience: Artists and those who like viewing artwork

Design & Style: Similar to social media platforms like DeviantArt and Pinterest and Instagram combined

CSS Framework: Bootstrap -- only bootstrap was occasionally used for the grid system and breakpoints

Color Palette

Primary background color, secondary background color, secondary background color, font color

Database

My database can't really be populated without users logging in and posting. I created an example user called user with email user@gmail.com and pw: User1!, so you can try logging and seeing the posts. All these posts were taken from DeviantArt or Pinterest.

Scope

Pages: Home Feed, Signup Page, Login Page, Art Submission Page, Main Page

Home Page
The home page has the tile page, a nav bar that let's you sign up or login, it also has an about section that covers the 3 main idea of Sharing, Discovering and Connecting. It also has a section to discover different topics, and a section of a featured posts.

Sign Up Page
This is where users can sign up and register, using email, password, and username

Login Page
Returning users can log in

Art Submission Page
Artists can post on this page, and it'll get uploaded onto their Main page.

Main Page
The main page is similar to instagram's main page, where your post is on the post tab, every post you've liked is on the liked tabs, and saved photos go into your saves tab. It also has your profile with follower and following count, with an about me section for you to edit.

HOW THE WEBSITE WORKS:
In order to post, you must first be logged in to the website. If you do not have an account, you can sign up. Then, it'll redirect you to a main page (main.php). This is where you'll see account info (followers, following, and about section). There is also a tab section (similar to instagram and other social media platform), where YOUR posts goes under the content tab, your liked posts will go under the hearts-tab, and your saved posts will go under the saved tab. Once you post, you can upload a file, write a title and description, along with choosing tags for your post. If successfully uploaded, it will be posted onto your contents tab. Here, you can delete or edit your post (CRUD).

Project Extras

1. Responsive Web Design I used bootstrap to ensure that boxes in a row lessen as dimensions get smaller, and to make the page look better when the dimensions get smaller

2. Event-Driven DOM This part is tied with PHP, where each post has HTML written, and deleting the post would technically delete the nodes.

3. Sessions/Cookies Each time a user logs in, a new session is created until they logout

4. File Upload Users are able to upload a file, and it will be posted
